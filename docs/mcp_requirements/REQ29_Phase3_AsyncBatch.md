# REQ-29 Phase3 异步与批量处理

## 背景
- 大量请求（例如多Action推理）需要异步/批量执行以提升吞吐。

## 目标
- Tool4/Tool5/Tool6提供异步接口与批量版本，客户端可轮询或订阅。

## 实施步骤
1. 设计Job管理器（状态：pending/running/done/failed）。
2. 提供批量API，一次提交多Action或多技能请求。
3. 实现通知机制（WebSocket/事件流）或轮询接口。
4. 更新客户端以支持异步结果展示。

## 验收标准
- 批量处理吞吐>单次的3倍，资源利用率提升。
- 异步API具备超时/取消机制。

## 风险
- 状态管理复杂 → 引入持久化队列并限制并发。

## 交付
- 作业管理模块、批量API文档、示例客户端。
