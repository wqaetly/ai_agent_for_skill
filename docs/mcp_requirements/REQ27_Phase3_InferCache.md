# REQ-27 Phase3 参数推理缓存

## 背景
- Phase3聚焦性能，需要为参数推理结果引入缓存以降低LLM调用成本。

## 目标
- 根据(skill_id, action_type, context_hash)缓存结果，支持TTL与失效策略。

## 实施步骤
1. 定义上下文哈希策略（描述、标签、已有Action、约束）。
2. 选型缓存介质（Redis/本地KV），实现读写与命中统计。
3. 设计失效策略（文件更新、规则变更时清理）。
4. 替换Tool4/ParameterInferencer中的直接计算流程，先查缓存再回源。

## 验收标准
- 缓存命中率目标≥60%，平均响应降低30%。
- 支持手动/自动清理与监控指标。

## 风险
- 哈希过细导致命中低 → 引入近似匹配或默认退化。

## 交付
- cache模块、配置模板、监控仪表、回归测试。
