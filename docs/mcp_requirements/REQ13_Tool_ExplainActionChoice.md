# REQ-13 MCP Tool: explain_action_choice

## 背景
- 需要向关卡/技能设计师解释推荐原因，增强信任与调试效率。

## 功能目标
- 输入recommended_action与skill_context，输出语义匹配、频率分析、模式识别、备选Action等说明。

## 实现步骤
1. 组合Tool3输出的推荐理由、Tool12统计与Tool11模式信息。
2. 生成结构化解释对象（语义/统计/模式/备选），供客户端渲染。
3. 当推荐基于业务规则或约束触发时，明确列出引用条目。

## 数据与依赖
- 依赖推荐结果、统计缓存、模式分析输出。

## 验收标准
- 每条解释至少包含两条证据（统计或示例）。
- 可选Action给出触发条件与适用场景。

## 风险
- 数据链断裂 → 在缺失字段时降级输出，并记录告警。

## 交付
- Tool实现、解释模板、可视化示例、对比测试报告。
